Dubbo支持RESTFul需要解决的问题

REST 协议核心要解决一个问题：Go 方法到 HTTP 接口的双向映射。

  • 普通 HTTP 调用 dubbo-go 服务，考虑的是 HTTP 到 Go 方法的映射
  • dubbo-go 服务调用 HTTP 服务，则是 Go 方法到 HTTP 接口的映射。

如何映射

映射内容
  • 路径映射
  • Header 处理（固定 Header 和 Header 值也是参数两种情况）
  • POST or GET or …（HTTP 方法映射）
  • 参数映射
映射规则

  • 接口

  • 映射配置

  • 《dubbo到rest的映射规则》

  • 没有现成网关吗？类似grpc-gateway
